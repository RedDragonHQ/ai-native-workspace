# inbox/ —— 草稿缓冲区

什么都可以先丢进来,以后再分类。

## 文件格式

文件名:`YYYY-MM-DD-简短描述.md`

内容:

```markdown
---
tags: [相关, 标签]
date: YYYY-MM-DD
---

# 标题

## 摘要
一段话概括发生了什么 / 学到了什么。

## 成就
- 做了事情 1
- 做了事情 2
- 交付了事情 3

## 产出文件
- `path/to/file1` —— 描述
- `path/to/file2` —— 描述

## 讨论要点
(可选)关键决策、悬而未决的问题、下次要回头看的点。
```

## 为什么这种格式?

- **frontmatter 标签**:以后 grep / 按主题筛用得上。
- **摘要**:强制一段话提炼 —— 比原始笔记有用。
- **成就**:是写简历的素材,要明确列出来。
- **产出文件**:带完整路径,让未来的你(或未来的 agent)能找到产物。
- **讨论要点**:捕获可能遗失的开放性问题。

## 不要放进 inbox 的东西

- **稳定的项目知识** → 直接放 `projects/<名称>/`
- **跨项目参考资料** → 放 `wiki/`(证明值得之后)
- **用户偏好 / 持久事实** → 放 CLAUDE.md 或项目文档

Inbox 是**缓冲区**,不是归档。
